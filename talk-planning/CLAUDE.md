# Collaboration Notes

## Project Identity

This project is a fixed-stage cinematic web presentation for faculty job talks and research talks.

It should feel like a live research argument, not a portfolio site, not a marketing page, and not a long-form visual essay.

## Design Preferences

- Use black as the default background.
- Use image/video backgrounds only when they clarify the argument or create useful atmosphere.
- Keep text short.
- Prefer fade, crossfade, and calm cinematic transitions.
- Avoid decorative cards unless representing actual artifacts such as papers.
- Avoid busy UI.
- Avoid long scrolling prose.
- Make the presentation feel speaker-led.

## Writing Preferences

- Avoid em dashes.
- Avoid "not X, but Y" phrasing.
- Avoid explanatory colons in polished prose.
- Avoid the word "concrete" unless explicitly requested.
- Avoid filler transition phrases.
- Prefer precise, serious, elegant language.

## Implementation Preferences

- Keep the app simple.
- Use scene data rather than hardcoding content into components.
- Use local assets.
- Use keyboard navigation as the primary presentation control.
- Keep scroll as a fallback.
- No presenter notes unless explicitly requested later.
- Web only is fine.

## Relationship To Existing Visual Essay

The current `my-anthropic-story` site is a reader-led visual essay.

This talk project should borrow useful patterns and assets from it, especially:

- full-screen media scenes
- paper card motif
- dark visual language
- video playback handling

But the talk should not inherit the long scrollytelling text rhythm.
